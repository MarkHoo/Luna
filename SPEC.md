# Luna - Llama.cpp 模型管理工具规格文档

## 1. 项目概述

**项目名称**: Luna
**项目类型**: 跨平台桌面应用
**核心功能**: 基于 llama.cpp 的模型下载、管理和推理启动工具
**目标用户**: 希望在本地运行大语言模型的用户

## 2. UI/UX 规格

### 2.1 布局结构

**窗口配置**:
- 主窗口尺寸: 1200x800 (最小 900x600)

## 3. 功能规格

### 3.1 核心功能

**模型管理**:
1. 列出所有可用模型（本地 + 可下载）
2. 从多个源下载模型
3. 删除本地模型
4. 显示模型详细信息

**模型下载源**:
| 源站点 | URL | 自动检测 |
|--------|-----|----------|
| Hugging Face | huggingface.co | ✅ |
| The Bloke (HF镜像) | huggingface.co/TheBloke | ✅ |
| ModelScope | modelscope.cn | ✅ (国内) |
| AI Collection | aicolle.com | ✅ (国内) |
| localai | localhost | 本地 |

**网络检测**:
- 启动时检测各源站点连通性
- 自动选择延迟最低的源作为默认
- 支持手动切换源

**模型启动**:
- 配置模型参数 (context size, threads, gpu layers)
- 启动 llama.cpp 推理服务器
- 集成 WebUI (如 Text Generation WebUI API)
- 实时显示日志输出

**系统集成**:
- 检测 GPU (CUDA/Metal/Vulkan)
- 显示内存使用情况
- 进程管理

### 3.2 用户交互流程

**首次使用**:
1. 启动应用 → 自动检测网络
2. 显示可用下载源
3. 引导用户下载第一个模型

**下载模型**:
1. 选择模型 → 选择下载源 → 开始下载
2. 显示实时进度
3. 下载完成 → 显示在已下载列表

**启动推理**:
1. 选择已下载模型
2. 配置参数
3. 点击启动
4. 自动打开浏览器访问 WebUI

### 3.3 数据结构

**本地模型配置** (`~/.luna/models.json`):
```json
{
  "models": [
    {
      "id": "llama-7b",
      "name": "LLaMA 7B",
      "path": "/path/to/model.gguf",
      "size": "13GB",
      "downloadedAt": "2024-01-01",
      "source": "huggingface"
    }
  ],
  "settings": {
    "defaultSource": "huggingface",
    "contextSize": 4096,
    "threads": 8,
    "gpuLayers": 99
  }
}
```

**下载任务**:
```json
{
  "id": "uuid",
  "modelId": "llama-7b",
  "source": "huggingface",
  "url": "https://huggingface.co/...",
  "status": "downloading",
  "progress": 45,
  "speed": "2.5MB/s"
}
```

### 3.4 边缘情况

- 下载中断 → 支持断点续传
- 网络超时 → 自动重试 3 次
- 磁盘空间不足 → 提前检测并提示
- 模型文件损坏 → 校验并提示重新下载
- GPU 不可用 → 回退到 CPU 模式

## 4. 技术架构

### 4.1 技术栈

- **框架**: Electron + React + TypeScript
- **状态管理**: Zustand
- **样式**: Tailwind CSS
- **构建**: Vite + electron-builder
- **后端通信**: IPC
